# 管理前端发布级 UI 修复设计

## 1. 背景与目标

现有管理前端已经具备真实接口、租户隔离、深浅主题和完整业务入口，但关键状态与视觉表达仍有四类发布风险：

1. 编辑页加载接口失败后继续展示初始化表单，用户可能把“读取失败”误判成“暂无配置”。
2. 通用弹窗的头部操作是可点击图标而不是按钮，小屏固定宽度也可能溢出。
3. 登录页的通用渐变光球与图片 Logo 缺少短剧 SaaS 产品辨识度。
4. 通用管理表格直接从工具栏开始，缺少页面标题、用途说明和必要的控件可访问名称。

本轮目标是在不改动接口、权限、租户作用域和认证流程的前提下，提高状态可信度、键盘与移动端可用性，并建立一致的管理页面层级。

## 2. 设计方向

### 2.1 状态可信度

- 编辑器只在数据加载成功后显示表单。
- 加载失败显示独立错误态和“重新加载”动作，不创建或展示伪空数据。
- 重试沿用原请求和路由参数；新建模式仍按原有初始化规则工作。
- 加载中、失败、成功三个状态互斥，避免页面闪烁或误操作。

### 2.2 弹窗

- 最大化、还原和关闭均使用语义化按钮，具备可访问名称和键盘焦点样式。
- 图标只承担视觉提示，对辅助技术隐藏。
- 调用方仍可传入原有宽度；组件统一限制最大宽度为视口宽度减去安全边距。
- 不改变现有 props、事件和 fullscreen 行为。

### 2.3 登录页

- 桌面使用“产品上下文 + 登录卡片”两栏布局，移动端收敛为单栏。
- 视觉签名来自短剧帧轨道、运行状态和真实能力说明，不使用泛化光球或虚构品牌。
- 文案只描述当前系统真实能力：代理商隔离、广告对账、版本发布控制。
- 保留主题、语言、验证码、记住登录和重定向逻辑。
- 用户名与密码字段补齐浏览器自动填充语义和可访问名称。

### 2.4 通用管理页

- 表格和配置页顶部显示当前页面标题、父级上下文与用途说明。
- 仅图标按钮提供明确名称，批量操作使用原生 `disabled` 状态。
- 表格全选、行选择、分页跳转和搜索输入提供可访问标签。
- 不重排业务菜单或修改页面路由，避免扩大本轮回归范围。

## 3. 非目标

- 不修改后端接口、权限判断、租户隔离或数据结构。
- 不引入新的 UI 依赖或字体资源。
- 不重构财务、风控等菜单信息架构。
- 不调整消费者 App UI。

## 4. 验收标准

- 编辑器接口失败时看不到可提交表单，可通过明确按钮重试。
- 弹窗操作可用键盘访问，320px 视口不产生横向溢出。
- 登录页在桌面和移动端都保持清晰层级，深浅主题可辨识。
- 通用管理页标题与说明来自现有 `pageConfig`，不重复维护文案。
- 新增契约测试、现有单元测试、类型检查与生产构建全部通过。
